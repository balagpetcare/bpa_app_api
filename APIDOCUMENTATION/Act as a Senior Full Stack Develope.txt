### 3. Current Progress (Completed Features)
We have successfully implemented the following APIs under `/api/v1`:

**A. Authentication (`/auth`)**
- **Register:** Creates a `User` and automatically creates a `UserWallet` (Balance: 0, Points: 0) inside a Prisma transaction.
- **Login:** Returns a JWT token.

**B. Pet Management (`/user/pets`)**
- **Register Pet:** Authenticated user can register a pet.
  - *Logic:* Validates input, creates Pet record, logs weight, adds vaccination record (if provided), and awards **50 Gamification Points** to the user's wallet using a transaction.
- **Get All Pets:** Fetches list of pets for the logged-in user with relations (AnimalType, Breed, ProfilePic).

**C. Media/File Handling (MinIO + Sharp)**
- **Upload Pet Photo:** (`POST /:petId/upload-photo`)
  - *Architecture:* Uses a separate `mediaController.js` and `uploadMiddleware.js` (Multer memory storage).
  - *Process:*
    1. Receives image via Multer.
    2. Compresses & Resizes (500x500, WebP, Quality 80) using `Sharp`.
    3. Uploads to MinIO (Local S3) using `@aws-sdk/client-s3`.
    4. Updates `Media` table and links it to `Pet` table in DB.
    5. Deletes old profile picture from S3 if exists.
  - *Config:* We use a central `src/config/appConfig.js` to manage dynamic Base URLs (e.g., handling IP changes for MinIO public links).

### 4. Database Schema (Key Models)

- **Pet:** (id, userId, name, animalTypeId, profilePicId...)
