📋 Project Technical Context (Copy & Save This)
Project Name: Bangladesh Pet Association (BPA) Super App Role: Backend Development

1. Tech Stack (Backend):

Runtime: Node.js

Framework: Express.js

Database: PostgreSQL (Relational Database)

ORM: Prisma (For schema management & migrations)

Authentication: JWT (JSON Web Tokens) & bcryptjs

Environment: dotenv for managing secrets

2. Architecture Pattern:

MVC Pattern: (Model-View-Controller) structure.

Folder Structure: Role-based separation.

src/routes/user/ vs src/routes/admin/

src/controllers/user/ vs src/controllers/admin/

Middleware: Custom authMiddleware for protecting routes.

3. Database Schema (Prisma Models):

User: id, name, email, password, pets[]

AnimalType: id, name (e.g., Dog, Cat), breeds[]

Breed: id, name, animalTypeId (Relation)

Pet: id, name, age, ownerId, animalTypeId, breedId (Uses relations, not strings)

4. Key Features Implemented:

Secure Auth: Registration & Login with Hashed Passwords.

Protected Routes: Profile & Pet Management protected by Bearer Token.

Cascading Dropdown Logic:

API to fetch AnimalTypes (Dog, Cat).

API to fetch Breeds based on AnimalTypeID (e.g., Select Dog -> Load German Shepherd).

Seeding: Automated script (seed.js) to populate initial data for all major animals in Bangladesh (Cattle, Poultry, Pets).

5. Future Roadmap (Frontend Architecture):

User App: Mobile App (Flutter) or Web (Next.js) - communicating via /api/v1/user

Admin Panel: Separate Next.js Application - communicating via /api/v1/admin

Both frontends will consume the same Backend API.

আপনার Bangladesh Pet Association (BPA) প্রজেক্টের বর্তমান অবস্থার ওপর ভিত্তি করে একটি প্রফেশনাল API Documentation তৈরি করে দিচ্ছি।

এই ডকুমেন্টেশনটি আপনার ফ্রন্টএন্ড ডেভেলপার (যে Flutter বা Next.js এ কাজ করবে) বা আপনার নিজের রেফারেন্সের জন্য ব্যবহার করতে পারবেন।

📘 BPA Backend API Documentation
Base URL: http://localhost:3000/api/v1 Authentication: Bearer Token (JWT)

🔐 1. Authentication (User)
ইউজার রেজিস্ট্রেশন এবং লগইনের জন্য।

A. Register New User
Method: POST

Endpoint: /user/register

Body (JSON):

JSON

{
  "name": "Gobinda Bala",
  "email": "gobinda@example.com",
  "password": "password123"
}
Success Response (201):

JSON

{
  "success": true,
  "message": "User registered successfully",
  "user": { ... }
}
B. Login User
Method: POST

Endpoint: /user/login

Body (JSON):

JSON

{
  "email": "gobinda@example.com",
  "password": "password123"
}
Success Response (200):

JSON

{
  "success": true,
  "message": "Login successful",
  "token": "eyJhbGciOiJIUzI1NiIsIn...",  // <--- এই টোকেনটি সেভ রাখতে হবে
  "user": { "id": 1, "name": "Gobinda Bala", "email": "gobinda@example.com" }
}
C. Get User Profile (Protected)
Method: GET

Endpoint: /user/profile

Headers: Authorization: Bearer <YOUR_TOKEN>

Response: লগইন করা ইউজারের তথ্য দেখাবে।

⚙️ 2. Settings (Dropdown Data)
পেট অ্যাড করার সময় ড্রপডাউন মেনু লোড করার জন্য।

A. Get All Animal Types (Dropdown 1)
সব ধরনের প্রাণীর লিস্ট (Dog, Cat, Bird) পাওয়ার জন্য।

Method: GET

Endpoint: /common/animal-types

Success Response:

JSON

{
  "success": true,
  "types": [
    { "id": 1, "name": "Dog" },
    { "id": 2, "name": "Cat" }
  ]
}
B. Get Breeds by Type (Dropdown 2)
নির্দিষ্ট প্রাণীর জাত (Breed) পাওয়ার জন্য।

Method: GET

Endpoint: /common/breeds/:typeId

Example URL: /common/breeds/1 (এখানে ১ হলো Dog এর আইডি)

Success Response:

JSON

{
  "success": true,
  "breeds": [
    { "id": 101, "name": "German Shepherd" },
    { "id": 102, "name": "Labrador" }
  ]
}
🐾 3. Pet Management
পেট অ্যাড করা এবং লিস্ট দেখার জন্য। (Login Required)

A. Add New Pet
Method: POST

Endpoint: /user/pets/add

Headers: Authorization: Bearer <YOUR_TOKEN>

Body (JSON): নোট: animalTypeId এবং breedId অবশ্যই সংখ্যা (Integer) হতে হবে।

JSON

{
  "name": "Rocky",
  "age": 2,
  "description": "Very friendly and active",
  "animalTypeId": 1,   // Dog (API 2.A থেকে প্রাপ্ত ID)
  "breedId": 101       // German Shepherd (API 2.B থেকে প্রাপ্ত ID)
}
Success Response (201):

JSON

{
  "success": true,
  "message": "Pet added successfully",
  "pet": { "id": 5, "name": "Rocky", ... }
}
B. Get All Pets
Method: GET

Endpoint: /user/pets/all

Headers: Authorization: Bearer <YOUR_TOKEN>

Success Response:

JSON

{
  "success": true,
  "pets": [
    {
      "id": 5,
      "name": "Rocky",
      "animalType": { "name": "Dog" },
      "breed": { "name": "German Shepherd" },
      "owner": { "name": "Gobinda Bala" }
    }
  ]
}
🚀 Frontend Workflow (কিভাবে ব্যবহার করবেন)
আপনার মোবাইল বা ওয়েব অ্যাপে এই স্টেপগুলো ফলো করবেন:

Step 1: ইউজার Login করবে -> অ্যাপ Token রিসিভ করে লোকাল স্টোরেজে সেভ করবে।

Step 2: Add Pet পেজে গেলে অ্যাপ প্রথমে /common/animal-types কল করে ১ম ড্রপডাউন পূর্ণ করবে।

Step 3: ইউজার যখন Dog সিলেক্ট করবে, অ্যাপ সেই Dog-এর ID নিয়ে /common/breeds/1 কল করে ২য় ড্রপডাউন পূর্ণ করবে।

Step 4: ফর্ম সাবমিট করার সময় অ্যাপ নাম বা টেক্সট না পাঠিয়ে শুধু ID (animalTypeId, breedId) এবং Token ব্যাকএন্ডে পাঠাবে।

এই ডকুমেন্টেশনটি সেভ করে রাখুন, এটি ডেভেলপমেন্টের সময় আপনার গাইডলাইন হিসেবে কাজ করবে।
